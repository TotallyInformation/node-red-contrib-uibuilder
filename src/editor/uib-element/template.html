With no msg.mode set, sending another message will automatically remove and replace the element.

    To remove an existing element, set msg.mode to "remove". This will also clear the cache.

    If node status is "Data Registered", when a new client connects to uibuilder, it will immediately be sent a copy of the data so that all connected clients have the same display.

    If you need to do more complex tasks such as update specific sub-elements, set the node to just output the configuration and then use the documentation to tweak it accordingly.

    NOTE: This node only works if you are using the new ESM or IIFE builds of the uibuilder front-end client library. It does not work with the previous uibuilderfe library.
